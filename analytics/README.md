# Аналитика EUROSFERA — отчёт на почту раз в день

Лёгкая своя аналитика для статического сайта на **SuperHosting.BG (cPanel + PHP)**.
Раз в день на почту приходит письмо: сколько визитов, с каких страниц, **откуда пришли**
(источник перехода) и **анонимизированный IP**. Без внешних сервисов, GDPR-friendly.

## Из чего состоит
| Файл | Что делает |
|------|------------|
| `_logger.php` | Ядро: пишет визит в SQLite-базу, **анонимизирует IP** (обнуляет хвост). |
| `px.php` | Трекинг-пиксель 1×1 для **статических .html** страниц. |
| `track.php` | Для PHP-страниц (если сайт на .php) — `include` в начале. |
| `daily_report.php` | Крон-скрипт: раз в день шлёт письмо и чистит старые записи (90 дней). |
| `.htaccess` | Закрывает базу и служебные файлы от посторонних. |

## Установка (5 шагов, ~15 минут)

**1. Залить папку** `analytics/` в `public_html` на хостинге (через cPanel → File Manager или FTP).

**2. Прописать почту** в `daily_report.php` (вверху):
```php
$TO   = 'a.tukan@euro.s.bg';   // куда слать отчёт
$FROM = 'site@euro.s.bg';      // СУЩЕСТВУЮЩИЙ ящик на хостинге (создай в cPanel → Email Accounts), иначе спам
```

**3. (рекомендуется) Вынести базу за пределы public_html.** В `_logger.php` поменяй:
```php
define('EURO_DB_PATH', '/home/ВАШ_ЮЗЕР/private/eurosfera_visits.sqlite');
```
Путь `/home/ВАШ_ЮЗЕР/` смотри в cPanel справа («Home Directory»). Если оставить по умолчанию —
база ляжет в `analytics/data/` и будет закрыта `.htaccess` (тоже ок, но «вне public_html» надёжнее).

**4. Вставить пиксель на каждую страницу.** Перед `</body>` каждой `.html` добавь:
```html
<img src="/analytics/px.php" width="1" height="1" alt="" style="position:absolute;left:-9999px" referrerpolicy="no-referrer-when-downgrade">
```
> Я могу вставить это во все 8 страниц автоматически — скажи, и сделаю.

**5. Настроить крон** (cPanel → **Cron Jobs**). Common Settings → «Once Per Day (0 0 * * *)», команда:
```
/usr/local/bin/php /home/ВАШ_ЮЗЕР/public_html/analytics/daily_report.php
```
(точный путь к php уточни в cPanel → «Select PHP Version» или у поддержки SuperHosting).

Проверить вручную можно открыв в браузере `https://твойдомен/analytics/daily_report.php` — придёт тестовое письмо.

## GDPR (обязательно для сайта в ЕС)
- IP **анонимизируется** автоматически (последний блок → 0), поэтому баннер-согласие для такой
  статистики, как правило, не требуется. Но честно укажи факт сбора в политике конфиденциальности.
- Срок хранения — **90 дней** (потом записи удаляются автоматически). Меняется в `_logger.php`
  (`EURO_RETENTION_DAYS`).
- Боты/краулеры отсекаются — в отчёте живые посетители.

## Что ещё есть «из коробки» на SuperHosting
- **cPanel → Metrics → AWStats** — готовая статистика со **странами**, браузерами, реферерами (графики).
- **cPanel → Metrics → Raw Access** — сырые логи Apache (включи «Archive logs», чтобы не стирались).

## Альтернатива без кода
- **Google Analytics 4** (бесплатно): вставь `G-XXXX` в `enhance.js` (поле `ga4_id`) → потом
  Looker Studio пришлёт ежедневный e-mail-отчёт по расписанию. Минус: внешний сервис + нужен баннер-согласие.
- **Matomo** (self-hosted на этом же хостинге) — полноценная аналитика как у GA, но на своём сервере.

> Рекомендация: начни с этого PHP-модуля (быстро и без внешних сервисов). Когда понадобятся
> страны/города и графики — добавим AWStats или Matomo.
